# HIL Self-Test Protocol and Pi Runner Spec

## Purpose
Define a hardware-in-the-loop (HIL) self-test flow that is safe by default and does not command motion or pressure changes. This document specifies protocol commands, result payload schema, and Raspberry Pi runner behavior.

## Scope and Safety
- Self-test profile in this spec is `SAFE` only.
- `SAFE` profile must not:
  - move any axis
  - home any axis
  - actuate syringe/pressure regulators
  - open/close pressure valves
  - pulse print/refuel outputs
- Allowed operations are read-only checks and comm/status validation.

## Protocol Commands (new)

### Command IDs (proposed, pending implementation)
- `0xFA` `CMD_SELFTEST_START` (host -> MCU)
- `0xFB` `CMD_SELFTEST_RESULT` (MCU -> host)
- `0xFC` `CMD_SELFTEST_DONE` (MCU -> host)
- `0xFD` `CMD_SELFTEST_ABORT` (host -> MCU)

These are new opcodes and must be added without changing existing framing/CRC rules:
- Frame format remains `[0xAA][LEN][PAYLOAD][CRC_LO][CRC_HI]`.
- TLV encoding remains little-endian and backward-compatible with existing protocol.

### `CMD_SELFTEST_START` payload (host -> MCU)
- Base: `[cmd=0xFA, seq8]`
- Optional TLVs:
  - `TAG_PROFILE` (`0x20`, len=1): `0x00=SAFE` (default)
  - `TAG_RUN_ID` (`0x21`, len=4): host-generated run identifier
  - `TAG_TIMEOUT_MS` (`0x22`, len=4): max run time; default 30000

### `CMD_SELFTEST_ABORT` payload (host -> MCU)
- Base: `[cmd=0xFD, seq8]`
- Optional TLV:
  - `TAG_RUN_ID` (`0x21`, len=4) to abort a specific run

### `CMD_SELFTEST_RESULT` payload (MCU -> host)
- Base: `[cmd=0xFB, seq8]`
- Required TLVs per emitted test case:
  - `TAG_TEST_ID` (`0x30`, len=2)
  - `TAG_NAME` (`0x31`, len=N UTF-8 bytes, max 32)
  - `TAG_PASS` (`0x32`, len=1, `0|1`)
  - `TAG_METRICS_JSON` (`0x33`, len=N UTF-8 bytes, max 96)
  - `TAG_TS_MS` (`0x34`, len=8, uint64 milliseconds since boot)
  - `TAG_RUN_ID` (`0x21`, len=4)

### `CMD_SELFTEST_DONE` payload (MCU -> host)
- Base: `[cmd=0xFC, seq8]`
- Required TLVs:
  - `TAG_RUN_ID` (`0x21`, len=4)
  - `TAG_TOTAL` (`0x35`, len=2)
  - `TAG_PASSED` (`0x36`, len=2)
  - `TAG_FAILED` (`0x37`, len=2)
  - `TAG_ABORTED` (`0x38`, len=1, `0|1`)
  - `TAG_TS_MS` (`0x34`, len=8)

## Result Schema

Pi-side JSON record schema for each result entry:

```json
{
  "test_id": 1001,
  "name": "comm_crc_known_vector",
  "pass": true,
  "metrics": {
    "latency_ms": 4,
    "rx_frames": 3
  },
  "timestamp": "2026-02-27T21:30:12.445Z"
}
```

### Field definitions
- `test_id` (integer): stable numeric identifier for this test case.
- `name` (string): stable machine-readable test name.
- `pass` (boolean): result for this case.
- `metrics` (object): key/value metrics (numbers/strings/bools), can be empty `{}`.
- `timestamp` (string): UTC ISO8601 time generated by Pi when result is received.

### Run-level JSON envelope

```json
{
  "run_id": 12345678,
  "profile": "SAFE",
  "started_at": "2026-02-27T21:30:10.102Z",
  "finished_at": "2026-02-27T21:30:13.017Z",
  "aborted": false,
  "summary": {
    "total": 6,
    "passed": 6,
    "failed": 0
  },
  "results": []
}
```

## SAFE Profile Test List (no motion/pressure)

Required SAFE tests (initial set):

1. `1001 comm_crc_known_vector`
- Validate firmware CRC routine against fixed known input.
- Metrics: `{ "crc": 19255 }`

2. `1002 comm_frame_roundtrip`
- Build and parse a local test frame; verify command/seq fields.
- Metrics: `{ "payload_len": 8 }`

3. `1003 status_frame_shape`
- Emit one status payload and validate command byte + minimum tag set present.
- Metrics: `{ "tag_count": 8 }`

4. `1004 uptime_counter_read`
- Read monotonic uptime/tick counter and verify progression.
- Metrics: `{ "delta_ms": 10 }`

5. `1005 flash_config_readonly`
- Read flash/imaging config values only (no trigger/pulse).
- Metrics: `{ "flash_delay_us": 0, "flash_width_ns": 0 }`

6. `1006 fw_build_info`
- Return compile metadata/version string and verify non-empty.
- Metrics: `{ "version_len": 14 }`

### Explicit exclusions in SAFE
- Any `CMD_MOVE_*`, `CMD_HOME_*`, gantry moves, gripper pump actuation.
- Any pressure regulator target changes, valve open/close, syringe reset/home.
- Any print/refuel pulse output.

## Raspberry Pi Script Contract

## CLI
Proposed script path:
- `pi/scripts/run_hil_selftest.py`

Proposed command:

```bash
python3 pi/scripts/run_hil_selftest.py \
  --port /dev/ttyACM0 \
  --baud 115200 \
  --profile SAFE \
  --timeout-ms 30000 \
  --out /var/log/labcraft/selftest/latest_selftest.json
```

## Invocation flow
1. Open serial link and perform normal handshake (HELLO/ACK if enabled).
2. Send `CMD_SELFTEST_START` with `TAG_PROFILE=SAFE`, `TAG_RUN_ID`, optional timeout.
3. Listen for streamed `CMD_SELFTEST_RESULT` frames.
4. For each result frame:
  - decode TLVs
  - map to JSON `{test_id,name,pass,metrics,timestamp}`
  - append to in-memory `results`
5. Stop on `CMD_SELFTEST_DONE` for matching `run_id`.
6. Write atomic JSON output:
  - write temp file in same directory
  - fsync
  - rename to final output path
7. Exit code:
  - `0` if done and `failed==0`
  - `2` if done and one or more tests failed
  - `3` on timeout, parse error, transport error, or aborted run

## JSON output file behavior
- Default output path:
  - `/var/log/labcraft/selftest/selftest_<run_id>.json`
- Optional symlink or copy:
  - `/var/log/labcraft/selftest/latest_selftest.json`
- Must include:
  - run envelope
  - ordered `results` array in receive order
  - raw protocol metadata for debugging (optional `raw` section)

## Abort Behavior
- Host may send `CMD_SELFTEST_ABORT` at any time.
- MCU should terminate current run and emit `CMD_SELFTEST_DONE` with:
  - `aborted=1`
  - `total/passed/failed` counts for completed tests
- Pi script should mark run `aborted=true` and return exit code `3`.

---

## Backward Compatibility
- Existing commands and frame parsing remain unchanged.
- Unknown self-test commands on older firmware should be tolerated by host script with a clear "unsupported firmware" error message.
- Host script should enforce protocol version gating before invoking self-test when available.
