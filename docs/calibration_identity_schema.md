# Calibration Identity Schema

## Purpose

Phase 2 introduces explicit registry-backed identity for:

- grouped reagents
- grouped printer-head types
- individual printer heads

This identity feeds the Phase 1 sidecar run context so later exact-pair and grouped-memory analysis can be built without changing historical file formats again.

## Identity Terms

### `stock_id`

- canonical identity for the actual stock solution loaded into the current printer head
- comes from the existing app stock-solution id source when available
- examples:
  - `Water_0.00_--`
  - app-specific stock ids generated by `StockSolutionManager._make_stock_id(...)`

### `reagent_id`

- grouped reagent identity for memory and analysis
- examples:
  - `water`
  - `glycerol_25pct`
  - `glycerol_50pct`

Multiple `stock_id` values can map to the same `reagent_id`.

### `printer_head_id`

- identity for one physical printer head instance
- examples:
  - `nozzle_100um_h03`
  - an explicit runtime serial if the registry has not been mapped yet

### `head_type_id`

- grouped nozzle/head-type identity
- examples:
  - `nozzle_80um`
  - `nozzle_100um`
  - `nozzle_120um`

Multiple `printer_head_id` values can map to the same `head_type_id`.

## Identity Quality Values

Allowed values:

- `explicit`
- `inferred`
- `unknown`

Legacy normalization:

- `derived` -> `inferred`
- `missing` -> `unknown`

## Registry Files

### `CalibrationMemory/entities/reagents.json`

```json
{
  "schema_name": "labcraft.calibration_memory.reagents_registry",
  "schema_version": 1,
  "updated_at_utc": "2026-03-06T00:00:00Z",
  "items": [
    {
      "reagent_id": "glycerol_25pct",
      "display_name": "25% Glycerol",
      "stock_ids": [],
      "aliases": ["glycerol_25pct", "25% glycerol"],
      "reagent_family": "aqueous_glycerol",
      "glycerol_percent": 25.0,
      "tags": ["baseline_study"],
      "notes": ""
    }
  ]
}
```

Required item fields:

- `reagent_id`
- `display_name`

Optional item fields:

- `stock_ids`
- `aliases`
- `reagent_family`
- `glycerol_percent`
- `tags`
- `notes`

## Run Context Identity Shape

Example explicit context:

```json
{
  "reagent_id": "glycerol_25pct",
  "reagent_display_name": "25% Glycerol",
  "reagent_family": "aqueous_glycerol",
  "glycerol_percent": 25.0,
  "stock_id": "Glycerol25_0.00_--",
  "printer_head_id": "nozzle_100um_h03",
  "printer_head_display_name": "100 um H03",
  "head_type_id": "nozzle_100um",
  "head_type_display_name": "100 um nozzle",
  "nominal_nozzle_diameter_um": 100.0,
  "measured_nozzle_diameter_um": 101.2,
  "identity_sources": {
    "reagent": "runtime_reagent_id",
    "printer_head": "runtime_printer_head_id",
    "head_type": "runtime_head_type_id"
  },
  "identity_quality": {
    "stock_id": "explicit",
    "reagent_id": "explicit",
    "printer_head_id": "explicit",
    "head_type_id": "explicit",
    "nominal_nozzle_diameter_um": "explicit",
    "measured_nozzle_diameter_um": "explicit"
  }
}
```

Example weak context:

```json
{
  "reagent_id": "water",
  "stock_id": "Water_0.00_--",
  "printer_head_id": "gripper_slot_2",
  "head_type_id": null,
  "identity_quality": {
    "stock_id": "explicit",
    "reagent_id": "inferred",
    "printer_head_id": "inferred",
    "head_type_id": "unknown",
    "nominal_nozzle_diameter_um": "unknown"
  }
}
```

## Schema Versioning

- all identity registry files start at `schema_version = 1`
- additive fields are allowed within version 1
- breaking changes require a version bump and migration path
- old run contexts should be normalized, not silently reinterpreted
